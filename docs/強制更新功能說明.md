# 強制更新功能說明

## 概述

KanaKana應用已集成強制更新機制，確保用戶始終使用最新版本的應用，以獲得最佳體驗和安全性。

## 功能特點

### 1. 自動版本檢查
- 應用啟動時自動檢查版本更新
- 支持iOS和Android平台
- 可配置檢查間隔

### 2. 強制更新
- 當檢測到需要強制更新時，阻止用戶使用舊版本
- 顯示美觀的更新提示界面
- 直接跳轉到應用商店進行更新

### 3. 可選更新
- 支持可選更新提示
- 用戶可選擇稍後更新或立即更新

### 4. 錯誤處理
- 網絡錯誤時的優雅處理
- 重試機制
- 離線模式下的降級處理

## 配置說明

### 1. app.json 配置
```json
{
  "expo": {
    "extra": {
      "versionCheck": {
        "enabled": true,
        "apiUrl": "https://your-api.com/api/version-check",
        "checkInterval": 3600000
      }
    }
  }
}
```

### 2. 版本號管理
- iOS: `buildNumber` 和 `CFBundleShortVersionString`
- Android: `versionCode` 和 `versionName`

## API 服務器

### 本地測試服務器
項目包含一個示例版本檢查服務器，位於 `scripts/version-check-server.js`

#### 啟動服務器
```bash
cd scripts
npm install
npm start
```

#### API 端點
- `POST /api/version-check` - 版本檢查
- `GET /health` - 健康檢查

#### 請求格式
```json
{
  "platform": "ios|android",
  "currentVersion": "1.0.0",
  "buildNumber": "1"
}
```

#### 響應格式
```json
{
  "isAvailable": true,
  "isRequired": false,
  "latestVersion": "1.1.0",
  "minRequiredVersion": "1.0.0",
  "updateUrl": "https://apps.apple.com/app/...",
  "releaseNotes": "更新內容..."
}
```

## 部署指南

### 1. 生產環境API
- 部署版本檢查API到生產服務器
- 更新 `app.json` 中的 `apiUrl`
- 配置適當的CORS設置

### 2. 版本發布流程
1. 更新應用版本號
2. 更新API服務器中的版本配置
3. 構建並發布應用
4. 測試版本檢查功能

### 3. 應用商店URL
更新以下文件中的應用商店URL：
- `src/utils/versionCheck.ts` - `getDefaultUpdateUrl` 函數
- API服務器配置

## 測試指南

### 1. 本地測試
1. 啟動版本檢查服務器
2. 修改 `app.json` 中的 `apiUrl` 為本地地址
3. 在服務器中設置較高的版本號
4. 重新啟動應用測試

### 2. 測試場景
- 正常版本檢查
- 強制更新場景
- 網絡錯誤處理
- 離線模式處理

## 注意事項

### 1. 版本號格式
- 使用語義化版本號 (Semantic Versioning)
- 格式：`major.minor.patch`
- 例如：`1.0.0`, `1.1.0`, `2.0.0`

### 2. 網絡安全
- 使用HTTPS進行API調用
- 考慮添加API認證機制
- 防止版本檢查被繞過

### 3. 用戶體驗
- 避免頻繁的版本檢查
- 提供清晰的更新說明
- 支持離線使用（不強制更新時）

## 故障排除

### 常見問題

1. **版本檢查失敗**
   - 檢查網絡連接
   - 確認API服務器狀態
   - 檢查API URL配置

2. **強制更新不生效**
   - 確認API返回的 `isRequired` 為 `true`
   - 檢查版本比較邏輯
   - 確認應用版本號正確

3. **應用商店跳轉失敗**
   - 檢查URL格式
   - 確認應用商店ID正確
   - 測試URL可訪問性
